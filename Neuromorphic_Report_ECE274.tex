\begin{abstract}
Spiking Neural Networks (SNNs) are attractive for edge inference because they are event-driven and sparse. In practice, however, energy savings are often limited by memory traffic and fixed-latency execution. This report presents a hardware-focused convolutional SNN (CSNN) flow that combines input-side dynamic gatekeeping with output-side early exit control. The gatekeeper suppresses low-information and redundant spike events before memory access, while an output confidence finite state machine terminates inference when class confidence is sufficient. Evaluated on N-MNIST, the proposed design achieves 91.83\% test accuracy (vs. 92.10\% baseline), rejects 30.5\% of input events, and reduces average SRAM fetches from approximately 172k to 52k per sample batch profile, corresponding to about 69\% lower memory-read activity. Inference commonly exits in 4--7 time steps instead of a fixed 20-step window. These results show that hardware-aware control logic can preserve accuracy while significantly reducing memory-dominated compute cost.
\end{abstract}

\section{Introduction}
Energy efficiency is the central constraint in practical neuromorphic deployment. Although SNNs are theoretically sparse, deployed systems still incur high cost when data movement and control are not explicitly co-designed with the model. In digital implementations, SRAM access and repeated updates of inactive state dominate power and latency budgets.

This project targets that gap through a hardware-first design objective: reduce memory-read activity without sacrificing useful classification accuracy. We start from a surrogate-gradient-trained CSNN baseline and introduce two architectural controls:
\begin{itemize}
    \item \textbf{Dynamic gatekeeping} to prevent low-value and redundant spikes from triggering downstream memory reads.
    \item \textbf{Early exit control} to stop temporal processing once class confidence is reached.
\end{itemize}

These mechanisms are evaluated using quantitative execution hooks aligned with an RTL-oriented interpretation of memory access and control signals. The report emphasizes hardware-relevant outcomes: activation sparsity, read traffic, and temporal execution length.

\subsection{Software-Hardware Co-Design Approach}
This project follows a full software-hardware co-design methodology. The algorithmic model is developed and trained in PyTorch, while each optimization is simultaneously expressed as a synthesizable Verilog RTL block. Hardware-proxy counters embedded in the PyTorch forward pass produce metrics (SRAM read-enable assertions, MAC clock-enable ratios, spike counts) that map directly to the control signals of the corresponding digital circuits. This tight coupling ensures that software-level accuracy and hardware-level efficiency are validated against the same dataflow, closing the gap between model research and deployment-ready circuit generation.

\section{Dataset and Preprocessing Pipeline}

\subsection{N-MNIST Event Stream}
N-MNIST converts static MNIST digits into asynchronous event streams captured by a dynamic vision sensor under controlled saccadic motion \cite{orchard2015converting}. Each event is represented as $(x,y,p,t)$ with pixel location, polarity, and timestamp.

\subsection{Spatial Normalization}
The native resolution of $34\times34$ is centered and cropped to $28\times28$ to align with a LeNet-style CSNN input. Events outside the retained field are discarded as peripheral noise.

\subsection{Temporal Binning}
Each sample is binned into $T=20$ time slices over a 300 ms horizon. Polarity is merged into a single input channel, yielding tensors of shape $(T,C,H,W)=(20,1,28,28)$. This representation preserves temporal structure while enabling batched training and hardware-style stepwise execution.

\section{Model and Learning Method}

\subsection{CSNN Topology}
The baseline network is a convolutional SNN with Leaky Integrate-and-Fire (LIF) neurons and rate-based output decoding. Temporal processing unfolds for up to $T=20$ steps.

\begin{figure}[htbp]
\centerline{\includegraphics[width=\linewidth]{sparse_snn_architecture.png}}
\caption{Hardware-aware CSNN pipeline with temporal unrolling and sparse event flow.}
\label{fig:architecture}
\end{figure}

\subsection{LIF Dynamics}
The membrane update at step $t$ is
\begin{equation}
V(t)=\beta V(t-1)+X(t)-S(t-1)V_{th},
\end{equation}
where $\beta$ is the leak term, $X(t)$ is synaptic input, and $S(t)$ is the output spike.
Spiking follows
\begin{equation}
S(t)=
\begin{cases}
1, & V(t)\ge V_{th}\\
0, & \text{otherwise.}
\end{cases}
\end{equation}
The implementation uses $\beta=0.9$ and nominal $V_{th}=1.0$.

\subsection{Surrogate Gradient Training}
Because the spike function is non-differentiable, training uses a fast-sigmoid surrogate derivative \cite{neftci2019surrogate,zenke2021remarkable}:
\begin{equation}
\frac{\partial S}{\partial V}=\frac{1}{\left(1+\alpha|V-V_{th}|\right)^2},
\end{equation}
with $\alpha=2.0$.

\section{Hardware-Oriented Optimizations}

\subsection{Dynamic Gatekeeping}
A gatekeeper block is placed ahead of Conv1 to reduce unnecessary memory reads. At each time step, the input event stream is evaluated before asserting memory or MAC activity.
\begin{itemize}
    \item \textbf{Importance monitor:} suppresses low-density spatial events likely to be background noise.
    \item \textbf{Burst filter:} removes immediate duplicate events that add little new temporal information.
\end{itemize}
When events are rejected, downstream read-enable behavior is suppressed, reducing SRAM access pressure.

Signal-level operation is summarized using the control relation
\begin{equation}
RE_{conv1}(t)=valid_{in}(t)\land gate\_keep(t)\land sys\_en(t),
\end{equation}
where $RE_{conv1}$ is the Conv1 SRAM read-enable. If $gate\_keep(t)=0$, no Conv1 weight fetch is issued, and the corresponding MAC lane remains idle for that cycle.

The per-cycle dataflow is:
\begin{equation}
\{RE_{conv1}, CE_{mac}\} \rightarrow \{V(t),S(t)\} \rightarrow score_c(t) \rightarrow terminate.
\end{equation}
This ordering makes the energy impact explicit: memory and switching activity are decided before neuron-state updates.

\subsection{Adaptive Threshold Regulation}
To prevent spike bursts in hidden layers, each neuron threshold is adapted over time:
\begin{equation}
V_{th}(t)=V_{th}(t-1)+\rho S(t)-\tau_{decay}.
\end{equation}
This raises threshold after firing and decays it gradually, encouraging sparse temporal encoding. In control terms, larger $V_{th}(t)$ lowers comparator hit probability and therefore reduces $S(t)=1$ events that would otherwise propagate to downstream memory requests in later layers.

\subsection{Confidence-Driven Early Exit}
A lightweight controller monitors output spike accumulation and terminates inference when confidence exceeds a threshold:
\begin{equation}
\textit{terminate}=\exists_{c\in C}\left(\sum_{t=0}^{t_{cur}} S_c(t)>\Psi_{conf}\right).
\end{equation}
The output counter update is
\begin{equation}
score_c(t)=score_c(t-1)+S_c(t),
\end{equation}
and once $terminate=1$, the controller sets
\begin{equation}
sys\_en(t+1)=0.
\end{equation}
This avoids unnecessary late time steps for easy samples and prevents additional memory accesses after confidence is reached.

\subsection{Cycle-Level Execution Example}
For one time step $t$, the control sequence is:
\begin{itemize}
    \item Evaluate $valid_{in}(t)$ and gatekeeper filters to generate $gate\_keep(t)$.
    \item Assert $RE_{conv1}(t)$ and $CE_{mac}(t)$ only if event and gate conditions are met.
    \item Update LIF states, produce spikes, and increment class counters.
    \item Check $terminate$; if asserted, deassert $sys\_en$ for the next cycle.
\end{itemize}
This sequence captures the intended RTL behavior and clarifies how control decisions translate to lower dynamic activity.

\subsection{Dynamic Gatekeeper Circuit (\texttt{dynamic\_gatekeeper.v})}
This block is the first-stage control gate and combines two subcircuits:
\begin{itemize}
    \item \textbf{Importance monitor} (\texttt{importance\_monitor.v}) for low-information event rejection.
    \item \textbf{Burst redundancy filter} (\texttt{burst\_redundancy.v}) for duplicate suppression.
\end{itemize}
It computes
\begin{equation}
gate\_keep = global\_enable \land spike\_valid \land imp\_keep \land corr\_keep,
\end{equation}
then directly drives
\begin{equation}
mem\_en = gate\_keep,\;\; mac\_en = gate\_keep.
\end{equation}
This logic creates a hard control boundary that prevents unnecessary SRAM and MAC switching.

\begin{figure}[htbp]
\centerline{\includegraphics[width=\linewidth]{Hardware_Architecture/circuit_dynamic_gatekeeper.png}}
\caption{Digital circuit diagram of the Dynamic Gatekeeper module showing the Importance Monitor and Burst Redundancy sub-blocks feeding a 4-input AND gate to produce the unified $gate\_keep$ control signal.}
\label{fig:circ_gatekeeper}
\end{figure}

\subsection{Importance Monitor Circuit (\texttt{importance\_monitor.v})}
The importance monitor implements a per-input-ID activity histogram using saturating counters:
\begin{itemize}
    \item Counter array: \texttt{cnt[0:NUM\_PRE-1]}.
    \item Window decay: periodic right shift every \texttt{WIN} cycles.
    \item Keep rule: \texttt{imp\_keep = (cnt[pre\_id] >= THRESH)}.
\end{itemize}
Functionally, this is a compact event-density estimator that filters low-frequency noise before memory access is requested.

\begin{figure}[htbp]
\centerline{\includegraphics[width=\linewidth]{Hardware_Architecture/circuit_importance_monitor.png}}
\caption{Importance Monitor circuit: saturating counter array with periodic window decay and threshold comparator driving $imp\_keep$.}
\label{fig:circ_imp}
\end{figure}

The burst filter tracks \texttt{last\_pre\_id} and \texttt{repeat\_count}. If the same presynaptic ID appears repeatedly beyond \texttt{K\_MAX}, then \texttt{corr\_keep} is deasserted for that cycle. The circuit is intentionally simple (register compare + small counter) to keep control overhead lower than saved downstream memory cost.

\begin{figure}[htbp]
\centerline{\includegraphics[width=\linewidth]{Hardware_Architecture/circuit_burst_redundancy.png}}
\caption{Burst Redundancy Filter circuit: register comparator with repeat counter to detect and suppress consecutive duplicate spike events.}
\label{fig:circ_burst}
\end{figure}

\subsection{Sparse Compute Path (\texttt{quantized\_sram.v} + \texttt{sparse\_mac.v})}
The compute datapath is event-gated:
\begin{itemize}
    \item \textbf{Quantized SRAM:} reads INT8 weights only when \texttt{re=1}.
    \item \textbf{Sparse MAC:} issues \texttt{read\_req} only on \texttt{spike\_in=1} and performs conditional accumulation.
\end{itemize}
Because SNN inputs are binary, multiplication is reduced to conditional addition, lowering arithmetic complexity and switching activity.

\caption{Quantized SRAM module: INT8 weight memory array with gated read-enable. When $re=0$, output is zeroed and no dynamic read power is consumed.}
\label{fig:circ_sram}
\end{figure}

\begin{figure}[htbp]
\centerline{\includegraphics[width=\linewidth]{Hardware_Architecture/circuit_sparse_mac.png}}
\caption{Sparse MAC circuit: conditional adder (no DSP multiplier needed) with spike-gated accumulation and SRAM read request generation.}
\label{fig:circ_mac}
\end{figure}

\subsection{Adaptive Neuron Circuit (\texttt{adaptive\_lif.v})}
The adaptive neuron contains three hardware primitives:
\begin{itemize}
    \item Leaky integrator via shift-add approximation.
    \item Comparator for spike generation (\texttt{is\_spike}).
    \item Threshold adaptation register updated by firing history.
\end{itemize}
This block enforces temporal sparsity in hardware by raising threshold after spikes and decaying toward baseline otherwise.

\begin{figure}[htbp]
\centerline{\includegraphics[width=\linewidth]{Hardware_Architecture/circuit_adaptive_lif.png}}
\caption{Adaptive LIF neuron circuit: leaky integrator with shift-add approximation, threshold comparator, soft-reset feedback, and adaptive $V_{th}$ register.}
\label{fig:circ_lif}
\end{figure}

\subsection{Early Exit Controller (\texttt{early\_exit\_fsm.v})}
The controller maintains class accumulators and a time-step counter. When confidence is met or $T_{max}$ is reached, it deasserts \texttt{sys\_enable} and asserts \texttt{done}. This creates a global clock-enable style termination signal shared by upstream blocks, allowing full-pipeline stop without requiring fixed-length execution.

\begin{figure}[htbp]
\centerline{\includegraphics[width=\linewidth]{Hardware_Architecture/circuit_early_exit_fsm.png}}
\caption{Early Exit FSM circuit: 10 class accumulators with confidence comparators and OR-reduction driving the global $sys\_enable$ shutdown signal.}
\label{fig:circ_fsm}
\end{figure}

\subsection{Top-Level Integration (\texttt{sparse\_snn\_top.v})}
The top module wires all blocks into a single control/data chain:
\begin{equation}
Input \rightarrow Gatekeeper \rightarrow SRAM \rightarrow Sparse\;MAC \rightarrow Adaptive\;LIF \rightarrow Early\ Exit\ FSM.
\end{equation}
This integration is the architectural basis for circuit generation per block and for end-to-end synthesis/implementation flows.

\section{Experimental Setup}
Training is performed in PyTorch using Adam ($10^{-3}$ learning rate), cosine annealing, and dropout in the readout stage. Hardware-proxy metrics are collected through forward hooks that count spike activity and estimate equivalent SRAM access events at each layer and time step. Specifically, we log proxies for $gate\_keep$, $RE_{conv1}$ activity ratio, per-layer spike rate, class-score trajectories $score_c(t)$, and inferred termination step to match the control/data signals in Table \ref{tab:signals}.

\section{Conclusion}
This work demonstrates that hardware-aware control in SNN inference can provide substantial practical efficiency gains beyond algorithmic sparsity alone. By combining input-side event filtering with output-side early exit, the design significantly lowers memory-read activity and temporal compute length while preserving classification performance on N-MNIST.

The primary contribution is not a new neuron model, but a deployment-oriented strategy that links SNN behavior to digital hardware costs. This makes the approach directly relevant to low-power neuromorphic accelerators and edge SoCs where memory access dominates energy.
